## Directory Purposes

**`pkg/provider/`:**
- Purpose: Terraform provider bootstrap and plugin configuration
- Contains: Provider schema, resource/datasource factory, authentication setup
- Key files: `provider.go`, `provider_helpers.go`, feature flags
- Generated files: None

**`pkg/resources/`:**
- Purpose: Terraform resource implementations with full CRUD lifecycle
- Contains: 150+ resource files (one per resource), helpers, state upgraders
- Key files: `<resource_name>.go` (e.g., `account.go`, `database.go`)
- Pattern: Resource functions return `schema.Resource` with Create/Read/Update/Delete
- Generated files: Marked with `// Code generated by...` comments

**`pkg/datasources/`:**
- Purpose: Read-only Terraform data source implementations
- Contains: 60+ datasource files, filtering logic, common helpers
- Key files: `<datasource_name>.go` (e.g., `database.go`, `accounts.go`)
- Pattern: Datasource functions return `schema.Resource` with Read context func only
- Generated files: None in this directory

**`pkg/sdk/`:**
- Purpose: Auto-generated Snowflake API client wrapping SQL execution
- Contains: Interface definitions, implementation, DTO builders, enums, validators
- Pattern: `<resource>s.go` (interface), `<resource>s_impl_gen.go` (implementation), `<resource>s_dto_gen.go` (data models)
- Generated files: Majority marked `_gen.go` from code generator
- Key files: `client.go`, `identifiers.go`, `datatypes/`
- Not to be edited manually except for `*_ext.go` files

**`pkg/schemas/`:**
- Purpose: Terraform Plugin SDK v2 schema definitions for resources and datasources
- Contains: Schema objects defining field properties, validation, computed fields
- Pattern: `<resource>_gen.go` (generated), `<resource>.go` (manual overrides/additions)
- Key files: `common_types.go` (shared schema fragments)
- Generated files: Most marked `_gen.go`

**`pkg/helpers/`:**
- Purpose: Shared utilities across resources and datasources
- Contains: Identifier encoding/decoding, random name generation, formatting helpers
- Key files: `resource_identifier.go`, `helpers.go`, `random.go`
- Not generated

**`pkg/acceptance/`:**
- Purpose: Helpers and fixtures for acceptance testing
- Contains: Test profile loaders, environment config, test data builders
- Subdirs: `bettertestspoc/` (newer testing patterns), `helpers/`, `testenvs/`
- Key files: `testprofiles/default.go`, `helpers/test_client.go`

**`pkg/testacc/`:**
- Purpose: Acceptance test suite requiring real Snowflake account
- Contains: Test setup, fixtures, individual acceptance test files
- Pattern: `*_acceptance_test.go` files matching test framework patterns
- Key files: `0_setup_test.go` (test context initialization), `0_setup_helper_funcs_test.go`
- Environment: Requires `TF_ACC=1` and Snowflake credentials

**`pkg/internal/`:**
- Purpose: Internal utilities not part of public API
- Subdirs: `provider/` (docs, validators), `collections/`, `util/`, `snowflakeenvs/`, `genhelpers/`
- Key files: `snowflakeenvs/env.go` (environment variable names)

**`pkg/manual_tests/`:**
- Purpose: Manual test scenarios and documentation
- Contains: Subdirectories per scenario (safe_delete, secondary_connection_promotion, etc.)
- Pattern: Terraform configurations and test instructions

**`docs/`:**
- Purpose: Provider documentation (generated)
- Contains: Resource and data source docs, guides
- Subdirs: `resources/`, `data-sources/`, `guides/`
- Note: Generated from templates and resource definitions

**`examples/`:**
- Purpose: Example Terraform configurations for users
- Contains: Resource examples, data source examples
- Pattern: `examples/<type>/<resource_name>/` directory structure

## Key File Locations

**Entry Points:**
- `main.go`: Terraform provider server initialization and protocol setup
- `pkg/provider/provider.go`: Provider schema and configuration factory
- `pkg/provider/resources/resources.go`: Resource name constants registry
- `pkg/provider/datasources/datasources.go`: Data source name constants registry

**Configuration:**
- `go.mod`: Go dependencies (terraform-plugin-sdk, gosnowflake, etc.)
- `Makefile`: Build targets, code generation, testing
- `.golangci.yml`: Linting configuration
- `pkg/internal/snowflakeenvs/env.go`: Environment variable names for Snowflake credentials

**Core Logic:**
- `pkg/sdk/client.go`: Main SDK client entry point
- `pkg/sdk/identifiers.go`: Identifier type definitions and parsing
- `pkg/resources/common.go`: Shared resource utilities and diff suppressors
- `pkg/resources/helpers.go`: Schema field processing helpers
- `pkg/resources/resource_helpers_create.go`: Create operation helpers
- `pkg/resources/resource_helpers_read.go`: Read operation helpers
- `pkg/resources/resource_helpers_update.go`: Update operation helpers

**Testing:**
- `pkg/testacc/0_setup_test.go`: Acceptance test context initialization
- `pkg/testacc/0_setup_helper_funcs_test.go`: Test fixture helpers
- `pkg/acceptance/bettertestspoc/assert/`: Custom assertion builders

**Code Generation:**
- `pkg/sdk/generator/`: SDK generator tool and definitions
- `pkg/sdk/generator/defs/`: Object and operation definitions
- `pkg/sdk/generator/gen/`: Generator implementation and templates

## Naming Conventions

**Files:**
- Resource implementation: `pkg/resources/<resource_name>.go` (kebab-case resource name)
  - Example: `account.go`, `account_parameter.go`, `api_integration.go`
- Resource schema: `pkg/schemas/<resource_name>_gen.go` (kebab-case, `_gen.go` for generated)
- DataSource implementation: `pkg/datasources/<datasource_name>.go`
  - Example: `database.go`, `account_roles.go`
- SDK operations: `pkg/sdk/<resource>s.go` (plural form for operation interface)
  - Example: `accounts.go`, `databases.go`, `schemas.go`
- Generated files: `*_gen.go` for auto-generated, `*_ext.go` for manual extensions
- Tests: `*_test.go`, `*_acceptance_test.go`
- State upgraders: `*_state_upgraders.go`

**Directories:**
- Resource groups: Flat structure under `pkg/resources/` with no subdirectories
- DataSource groups: Flat structure under `pkg/datasources/` with no subdirectories
- SDK by object type: Flat structure under `pkg/sdk/` with pattern `<objects>*.go`
- Test utilities: `pkg/testacc/` flat, `pkg/acceptance/` with logical subdirs

**Types:**
- Resource functions: PascalCase, return `*schema.Resource`
  - Example: `Account()`, `Database()`, `Schema()`
- CRUD functions: PascalCase with operation prefix
  - Example: `CreateAccount()`, `ReadAccount()`, `UpdateAccount()`, `DeleteAccount()`
- DataSource functions: PascalCase, same naming as resources
  - Example: `Database()`, `Databases()`
- SDK interfaces: PascalCase plural
  - Example: `Accounts`, `Databases`, `Schemas`
- Options structs: PascalCase suffix with Operation name
  - Example: `CreateAccountOptions`, `AlterDatabaseOptions`, `ShowSchemaOptions`
- Identifiers: PascalCase suffix with "Identifier"
  - Example: `AccountObjectIdentifier`, `DatabaseObjectIdentifier`, `SchemaObjectIdentifier`

## Where to Add New Code

**New Resource:**
- Primary code: `pkg/resources/<resource_name>.go`
  - Define schema in `var <resource_name>Schema = map[string]*schema.Schema{}`
  - Implement Create/Read/Update/Delete functions
  - Return `*schema.Resource` with schema and operation handlers
- Schema (if not auto-generated): `pkg/schemas/<resource_name>.go` or `<resource_name>_gen.go`
- Register resource: Add to `pkg/provider/resources/resources.go` constants and `getResources()` map in `pkg/provider/provider.go`
- Tests: `pkg/testacc/resource_<resource_name>_acceptance_test.go`

**New DataSource:**
- Primary code: `pkg/datasources/<datasource_name>.go`
  - Define schema in `var <datasource_name>Schema = map[string]*schema.Schema{}`
  - Implement Read function with query logic
  - Return `*schema.Resource` with schema and read handler
- Schema: Usually manual in datasource file, not in `pkg/schemas/`
- Register datasource: Add to `pkg/provider/datasources/datasources.go` constants and `getDataSources()` map
- Tests: `pkg/testacc/data_source_<datasource_name>_acceptance_test.go`

**Utilities/Helpers:**
- Shared identifier helpers: `pkg/helpers/resource_identifier.go`
- Shared resource helpers: `pkg/resources/helpers.go` or `resource_helpers_*.go`
- Validators: `pkg/internal/provider/validators/`
- Test helpers: `pkg/acceptance/helpers/`

**New SDK Operation (rarely done manually):**
- Extend: `pkg/sdk/<resource>s_ext.go` file with `// Adjusted manually` comments
- Never regenerate entire SDK - use `make generate-sdk SF_TF_GENERATOR_ARGS="--filter-object-names=<ObjectName>"`
- Reference: `pkg/sdk/generator/defs/` for object definition format

## Special Directories

**`pkg/schemas/`:**
- Purpose: Generated schema definitions
- Generated: By `make generate-show-output-schemas` from SDK Show output
- Committed: Yes, schemas are committed to git
- Editing: Only `*_gen.go` files are auto-generated; manually edit `*.go` files without `_gen` suffix

**`pkg/sdk/`:**
- Purpose: Generated SDK client code
- Generated: By `make generate-sdk` from definitions in `pkg/sdk/generator/defs/`
- Committed: Yes, SDK code is committed to git
- Editing: Only edit `*_ext.go` files manually; never edit `*_gen.go` files directly

**`pkg/testacc/`:**
- Purpose: Acceptance tests requiring real Snowflake account
- Generated: None - all manual tests
- Committed: Yes, tests are committed to git
- Cleanup: Not committed: test artifacts like `*.tfplan`, `*.tfstate`

**`docs/`:**
- Purpose: Generated documentation from templates and resource definitions
- Generated: By `make docs` from templates in `templates/`
- Committed: Yes, docs are committed to git
- Editing: Never edit docs directly; edit resource descriptions or `templates/` instead

**`internal/`:**
- Purpose: Internal tracking, utilities
- Generated: None
- Committed: Yes
- Visibility: Not exported outside package tree

## Resource ID Encoding

**Format:** Components joined by `.` separator (not deprecated `|`)

**Examples:**
- Account object: `"my_database"`
- Database object: `"my_database.my_schema.my_table"`
- Schema object: `"my_database.my_schema"`

**Encoding:** `helpers.EncodeResourceIdentifier(objectIdentifier.FullyQualifiedName())`

**Decoding:** `sdk.Parse<Type>ObjectIdentifier(d.Id())` returns typed identifier with component getters

## Code Generation Workflow

**Show Output Schemas:**
```bash
make generate-show-output-schemas
```
- Generates: `pkg/schemas/*_gen.go` from SDK SHOW operation results
- Committed: Yes

**SDK Code:**
```bash
make generate-sdk SF_TF_GENERATOR_ARGS="--filter-object-names=NetworkRules"
```
- Generates: `pkg/sdk/<objects>*.go` from definitions
- Scope: Specific objects via `--filter-object-names` parameter
- Never: Use bare `go generate` - regenerates all SDK objects

**Documentation:**
```bash
make docs
```
- Generates: Files in `docs/resources/`, `docs/data-sources/`
- Source: Terraform plugin documentation generator from resource schemas
- Never: Edit generated docs files directly
